# GVA Skill Usage

This guide explains how to use the `oke-gva-deployer` skill to add a Generic VNIC Attachment (GVA) node pool to an OKE cluster.

## Quick Start (Recommended)
Run the guided menu script:

```bash
./scripts/gva-menu.sh
```

The menu:
- Uses the **cluster name** you enter (defaults to `cluster3`).
- Pulls cluster OCID from `~/.kube/config` if available.
- Uses region/tenancy defaults from `/Users/chipinghwang/.oci/config`.
- Auto-discovers cluster details (compartment, k8s version).
- Lists VCNs, then subnets, then OKE images for your k8s version.
- Prompts only for values it cannot discover.
- Prints a ready-to-run OCI CLI command and a test Deployment manifest.

## Manual Flow (No Menu)
If you can’t run the menu, follow this sequence:

1. **Identify cluster OCID**
   - From kubeconfig: `~/.kube/config` contains `--cluster-id <ocid>` in the user exec args.
2. **Fetch cluster details**
   - `oci ce cluster get --cluster-id <cluster_ocid> --region <region>`
3. **List VCNs**
   - `oci network vcn list --compartment-id <compartment_ocid> --region <region>`
4. **Select a VCN and list subnets**
   - `oci network subnet list --compartment-id <compartment_ocid> --vcn-id <vcn_ocid> --region <region>`
5. **Pick OKE image for the cluster’s k8s version**
   - `oci ce node-pool-options get --node-pool-option-id all --region <region>`
6. **Create node pool with GVA**
   - Use the `oci ce node-pool create` command generated by the skill.
7. **Verify**
   - `kubectl describe node <node_name>` for GVA extended resources and taint.

## Inputs You Still Need to Provide
Even with discovery, you must choose:
- Node pool name
- Node shape (and OCPUs/memory for Flex)
- Node count
- Node placement subnet (from VCN list)
- GVA VNIC profiles (subnet, applicationResource, ipCount)
- Image (from the menu list)

## Test Deployment
The menu prints a test Deployment manifest. Replace `<image>` with a real image and apply:

```bash
kubectl apply -f gva-test.yaml
```

Ensure the pod requests **exactly one** Application Resource and includes the required toleration.

## Troubleshooting
- **Pods Pending**: Check resource name, toleration, and IP capacity.
- **Validation errors**: Pods must request exactly 1 unit of a single Application Resource.
- **No extended resources on node**: GVA was not enabled on the node pool.
